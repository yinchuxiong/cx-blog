# Agent 记忆范式笔记

Agent 的记忆不能简单理解成“存历史聊天记录”，它更像是一套分层系统。不同记忆类型的区别，核心不在于存储形式，而在于它们分别服务什么能力：理解当前任务、保存知识、回顾经历、沉淀经验、执行流程、维护状态、形成个性化协作。

## 1. 上下文记忆

上下文记忆就是当前会话里的工作现场。

它包括当前对话内容、正在处理的任务、刚刚读取的信息、工具执行结果、临时推理过程等。它的特点是短期、临时、和当前任务强相关。

可以理解为：

> 当前这一轮我在干什么，以及刚刚发生了什么。

## 2. 语义记忆

语义记忆记录的是相对稳定的事实、知识和概念。

比如用户信息、角色设定、项目背景、概念定义、文档知识、知识库内容等。它通常适合用文档、数据库、向量知识库、知识图谱来存储和检索。

它的核心是：

> 什么是真的，什么是稳定知识。

## 3. 情节记忆

情节记忆记录的是过去发生过的具体事件和经历。

比如某次任务怎么处理、某次报错怎么解决、某次用户做过什么选择、某次交互中出现了什么反馈。它更像日志或者日记。

它的特点是数量最多、最原始，但也最容易有噪声，因为单次经历不一定能代表普遍规律。

可以理解为：

> 过去具体发生过什么。

## 4. 反思记忆

反思记忆是从情节记忆中提炼出来的经验判断。

它不是简单记录事件，而是从多个事件或一次重要经历中总结出“以后应该如何判断”。比如发现某个用户更喜欢先讲直觉再讲定义，或者某个项目里登录问题经常和 token 生命周期有关。

它的核心是：

> 从过去经历中学到了什么规律。

## 5. 程序性记忆

程序性记忆记录的是可重复执行的方法流程，也就是“怎么做”。

比如排查登录问题的步骤、发布流程、写文章流程、代码审查流程、某个工具的使用方法等。它通常比反思记忆更稳定、更少，也更适合直接指导行动。

它可以来自反思沉淀，也可以来自文档、规范、专家经验。

可以理解为：

> 遇到某类问题时，应该按什么步骤做。

## 6. 状态记忆

状态记忆记录的是当前任务、环境或世界的实时状态。

比如任务做到哪一步、哪些文件改了、哪些测试跑过、哪些事项待处理、当前流程处于哪个节点。它更像任务白板，而不是历史日志。

它和情节记忆的区别是：

> 情节记忆记录“做过什么”；状态记忆记录“现在到哪了”。

## 7. 个性化 / 关系记忆

个性化记忆记录的是如何更好地和某个用户协作。

比如用户偏好的表达方式、长期关注方向、沟通习惯、禁忌、常用工具、希望助手主动还是保守等。它可能来源于语义记忆、情节记忆和反思记忆，但最终服务的是个性化协作。

它的核心是：

> 以后如何更懂这个人、配合这个人。

## 三类最容易混的记忆

情节记忆、反思记忆、程序性记忆之间可以这样区分：

> 情节记忆是原始经历。  
> 反思记忆是从经历里提炼出的经验判断。  
> 程序性记忆是经过验证后沉淀成的可执行流程。

例如：

- 情节记忆：上次登录失败，原因是 token 过期。
- 反思记忆：这个项目的登录问题经常和 token 生命周期有关。
- 程序性记忆：排查登录问题时，依次检查 cookie、token 过期时间、刷新接口、鉴权中间件。

所以它们大致是一个从原料到方法的链条：

```text
情节记忆 -> 反思记忆 -> 程序性记忆
```

情节记忆最多，但不一定准；反思记忆更少，是经验判断；程序性记忆最少，但行动价值最高。不过程序性记忆也不一定永远正确，因为流程可能过时，所以最好给记忆加上来源、时间、置信度、适用范围和是否被验证过等元信息。

## 最简总结

- 上下文记忆：当前任务现场。
- 语义记忆：稳定事实和知识。
- 情节记忆：过去发生过的具体经历。
- 反思记忆：从经历中总结出的经验判断。
- 程序性记忆：可复用的执行流程。
- 状态记忆：当前进度和环境状态。
- 个性化记忆：如何更好地配合某个用户。

一句话概括：

> Agent 记忆系统的关键，不是把所有信息都存下来，而是区分哪些是当前上下文，哪些是稳定知识，哪些是历史经历，哪些可以抽象成经验，哪些值得固化成可执行流程。
